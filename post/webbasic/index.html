Web程序运行原理

基本原理

  • 运行原理
    • 用户打开浏览器，输入URL地址
    • 客户端浏览器通过HTTP协议向服务器端发送浏览请求
    • 服务器端通过CGI程序接受请求，通过http协议把得到的数据返回给客户端浏览器
    • 客户端浏览器解释并显示HTML页面
  • DNS
    • 域名系统（Domain Name System）将主机名和域名转换为IP地址。
    • DNS解析过程
      • 用户输入URL，浏览器抽出域名字段（要访问的主机名）并将这个主机名传送给DNS应用程序的客户端
      • DNS客户端向DNS服务器发送一份查询报文，包含要访问的主机名字段
      • DNS服务器向DNS客户端发送一份回答报文，包含主机名和对应IP地址
      • 浏览器接收到来自DNS的IP地址，向IP地址定位的HTTP服务器发起TCP连接
  • HTTP（Hyper Text Transfer Protocol）
    • 请求行
      • 请求方法
        • POST：参数通过表单传输，数据包在请求体 增
        • GET ：参数显式URL（常用 查
        • PUT：改
        • DELETE：删
          登陆时用POST，防止账号密码泄露
      • URI
      • HTTP协议
    • 请求头
      • Cookie
      • Referer
      • User-Agent 等
    • 请求体 在HTTP请求中传递数据的实体，常用语POST和PUT
    • 响应
      • 响应状态码
      • 响应头
        • Allow
        • Content-Type
        • Server等
      • 响应体 HTTP请求返回的内容，响应的正文数据都在响应体中
        浏览器要解析的内容就是响应体
  • URI 与 URL
    • URI（Uniform Resource Identifier）统一资源标识符
      • 资源命名机制
      • 存放资源的主机名
      • 资源自身的名称
    • URL（Uniform Resource Locator）统一资源定位符
      主要用在各种万维网客户端程序和服务器端程序中
      • URI的一个子集
      • 协议（服务方式）
      • 目的主机IP地址（端口号）
      • 主机资源的具体机制
    • URN（Uniform Resource Name）统一资源名 URN是URL的一种更新形式
    • URI URL URN 三者关系 URI是URL更高层次的抽象。URI和URL都只定义了资源是什么，URL定义了如何访问资源，还提供了定位资源的信息。
  • HTTPS
    • 在HTTP的基础上通过加密和身份认证保持了传输过程的安全性
    • TLS（传输层安全性协议）和前身SSL（安全接套字层）为互联网通信提供安全以及数据完整性保障
    • 采用SSL/TLS，HTTP就拥有了HTTPS的加密、证书和完整性保护这些功能
    • HTTP经过了加密处理、认证和完整性保护后就是HTTPS
  • Web应用程序的组成
    • 处理器 把客户端发送过来的http请求接受并处理，徽县调用模板引擎，将模板引擎生成的HTML文档通过http协议返回给客户端
      通常处理器会接受HTTP请求，然后解析路由，最后将URL映射到对应的控制器中
    • MVC（Model、View、Contorller）
    • 模板引擎 用于将模板和数据组合在一起，最终生成HTML文档，通过HTTP响应报文发送给客户端
      实现界面和数据分离
