操作系统 Chapter2

前趋图和程序执行 P32

  • 前趋图

    • I：输入操作
    • C：计算操作
    • P：打印操作

  • 程序顺序执行

    • 程序顺序执行
    • 程序顺序执行时的特征
      • 顺序性：严格按照规定的顺序执行
      • 封闭性：程序运行时独占全机资源
      • 可再现性：只要环境和初始条件相同 结果相同
        从头到尾不停顿的执行

  • 程序并发执行

    • 程序的并发执行 不存在前趋关系，可以并发执行
    • 程序并发执行时的特征
      • 间断性
        因为为完成同一项任务而互相合作，共享系统资源，所以执行期间形成了相互制约的关系，导致程序会有“执行-暂停-执行”的规律，“走走停停”
      • 失去封闭性 当存在多个可以并发执行的的程序时，系统中的各种资源将为他们所共享，资源状态也会因为这些程序改变，
      • 不可再现性

进程的描述 P35

  • 进程的定义和特征

    • 进程的定义
      为了对并发执行的程序加以描述和控制，引入了“进程”概念
      为了让并发执行的每个程序都能独立的运行，引入了专门的数据结构：进程控制块 PCB （Process Control Block）
      （1）进程是程序的一次执行
      （2）进程是一个程序及其数据在处理机上顺序执行所发生的活动
      （3）进程是具有独立功能的程序在一个数据集合上运行的过程，他是系统进行资源分配和调度的一个独立单位
      进程是进程实体的运行过程

    • 进程的特征

      • 动态性：
        是进程实体的执行过程
        创建而产生 调度而执行 撤销而消亡

      • 并发性
        多个进程实体同存于内存中
        没有建立PCB的程序是不能并发执行的

      • 独立性
        在传统的OS中，独立性指的是：进程实体是一个能独立运行，独立获得资源和独立接受调度的基本单位。凡是未建立PCB的程序是不能作为一个独立的单位参与运行

      • 异步性 为使进程在并发执行时虽然具有异步性，但依旧能保证进程并发执行的结果是可以再现的，配置了进程同步机制

  • 进程的基本状态及转换

    • 进程的三种基本状态

      • 就绪状态（Ready） 已经被分配到除了CPU以外的所有必要资源 如果有许多处于就绪状态的进程，就会产生就绪队列

      • 执行状态（Running） 已经获得CPU 单处理机 只有一个进程处于执行状态 多处理机 会有多个进程属于执行状态

      • 阻塞状态（Block） 因为某事件暂时无法继续执行时的状态（如申请缓冲区失败？） OS会把处理机分配给另一个就绪进程，让受阻的进程处于暂停状态 阻塞态的进程也会排成阻塞队列

    • 三种基本状态的转换
      就绪->执行：进程调度 分配CPU
      执行->就绪：时间片完 被剥夺处理机
      执行->阻塞：I/O请求 （如进程需要访问某正在被其他进程访问的临界资源）
      阻塞->就绪：I/O完成

    • 创建状态和终止状态

      • 创建状态 申请一个空白PCB -> 填写用于控制和管理进程的信息
        如果系统没有足够的内存装入进程 进程无法被调度执行
        此时就是创建状态
      • 终止状态 删除进程 -> PCB清零 -> 把PCB归还给系统
        当进程到了一个自然结束点，或者是出现了无法克服的的错误，或者被操作系统终结，就会进入终止状态

  • 挂起操作和进程状态的转换

    用于某个进程，和进程会被挂起，进程暂时不接受调度，对应激活操作

    • 挂起操作的引入

      基于系统和用户的如下需要

      • 终端用户的需要
      • 父进程请求
      • 负荷调节的需要
      • 操作系统的需要

    • 引入挂起原语操作后三个进程状态的转换

      挂起原语：Suspend 激活原语：Active

      • 活动就绪 -> 静止就绪
      • 活动阻塞 -> 静止阻塞
      • 静止就绪 -> 活动就绪
      • 静止阻塞 -> 活动阻塞

    • 引入挂起操作后五个进程状态的转换

  • 进程管理中的数据结构

    操作系统对进程运行情况、资源使用等信息的组织和维护是通过建立各种数据结构的方式来实现

    • 操作系统中用于管理控制的数据结构
      • 内存表
      • 设备表
      • 文件表
      • 用于进程管理的进程表（PCB）
    • PCB的作用 为了便于系统描述和管理进程的运行
      • 作为独立运行基本单位的标志（能取得OS服务
      • 能实现间断性运行方式（恢复CPU现场
      • 提供进程管理所需要的信息（在内外存的指针
      • 提供进程调度所需要的信息（比如状态信息
      • 实现与其他进程的同步与通信（信号量机制
    • 进程控制块中的信息
      • 进程标识符
        唯一的标识一个进程
        • 外部标识符 方便用户/进程对进程的访问
        • 内部标识符 方便系统对进程的使用
      • 处理机状态
        由处理机中各种寄存器的内容组成
      • 进程调度信息
        状态、优先级、进程调度所需的其他信息、事件（阻塞原因）
      • 进程控制信息
        程序和数据的地址、进程同步和通信机制、资源清单、链接指针
    • 进程控制块的组织方式
      • 线性方式
        系统中所有的PCB组织在一张线性表中 实现简单开销小，但每一次都要扫描整张表，适合进程不多的系统
      • 链接方式
        通过PCB中的链接字链接成一个队列，形成就绪队列、阻塞队列、空闲队列
      • 索引方式
        索引表

    • 引起进程终止的事件
      • 正常结束
      • 异常结束
      • 外界干预
    • 进程的终止过程
      • 根据被终止进程的标识符，从PCB集合中检索出该进程的PCB，读出进程状态
      • 如果正处于执行状态，终止执行，置调度标志为真，用于指示被终止后应该重新调度
      • 如果有子孙进程，予以终止
      • 被终止的进程资源全部还给父进程，或者系统
      • 将PCB从队列中（或链表）中移除，等待其他程序收集信息

进程同步 P47

  • 进程同步的基本概念

    • 两种形式的制约关系

      • 间接相互制约关系（临界资源
      • 直接相互制约关系 （输入 计算的缓冲区共享

    • 临界资源

      把counter当成临界资源处理 要让生产者和消费者进程互斥地访问变量counter

    • 临界区

      把每个进程中访问临界资源的代码称为临界区 每个进程进入临界区之前需要对临界资源检查 看是否在被使用 需要在临界区之前加上一段进入区 entry section 后面加入退出区 exit section 其他部分都叫做剩余区

  • 硬件同步机制

    • 关中断
      • 滥用会导致严重后果
      • 时间过长会影响系统效率 限制处理器
      • 不适用于多CPU 因为一个处理器上关中断并不能防止进程在其他处理器上执行相同的临界段代码
    • 利用Test-and-Set指令实现互斥
    • 利用Swap指令实现进程互斥
