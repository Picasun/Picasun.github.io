操作系统 Chapter1

操作系统的目标和作用 P1

  • 操作系统的目标

    • 方便：不用直接在硬件运行 机器可以编译命令将用户写的程序翻译成机器代码 方便用户操作计算机系统

    • 有效性：1.提高系统资源利用率 （软硬件） 2.合理组织工作流程

    • 可扩展性：微内核结构能方便增添新的功能和模块

    • 开放性：软硬件兼容 方便实现互联

    • 扩展性

  • 操作系统的作用

    • 作为用户和计算机硬件系统之间的接口

    • 计算机系统资源（处理机、存储器、IO、文件）的管理者

    • 实现对计算机资源的抽象

      • I/O设备管理软件对计算机硬件操作的第一层抽象

      • 文件管理软件实现了第二层抽象

  • 操作系统发展的动力

    • 不断提高计算机资源利用率

    • 方便用户

    • 器件的更新换代

    • 计算机体系结构的不断发展

    • 新的应用需求

操作系统的发展过程 P5

  • 未配置的操作系统

    • 人工操作：纸带机器

      • 资源空闲

      • 用户独占全机器

    • 脱机输入/输出方式：输入到磁盘上 需要时调入内存

      • 减少CPU空闲

      • 提高I/O速度

  • 单道批处理系统

    • 脱机方法（把作业输入磁带）➕监督程序Monitor 监督程序给予控制权给作业

    • CPU利用率降低

  • 多道批处理系统

    • 设计基本概念：进一步提高资源利用率和系统吞吐量

      • 1.作业放在外存 形成后备队列

      • 2.按照算法调入内存 共享CPU等资源

    • 优缺点：

      • 1.资源利用率高（多道程序交替进行） 2.系统吞吐量大（资源始终“忙碌”）

      • 1.平均周转时间长（要排队） 2.无交互能力（交给系统后不能交互）

    • 需要解决的问题

      • 处理机争用：要满足各程序运行 也要提高处理机利用率

      • 内存分配保护 不会因某程序异常破坏其他

      • I/O 设备分配问题

      • 文件组织和管理问题

      • 作业管理问题

      • 用户与系统接口问题

  • 分时系统

    • 分时系统的引入

      • 希望 人机交互

      • 希望 共享主机

    • 实现中的关键问题

      • 及时接受：

        • 设置多路卡 （xx个终端就配置一个xx用户的多路卡）

      • 及时处理：

        • 直接进内存

        • 轮转运行方式（时间片：一个很短的时间 运行完立即调度下一个作业）

    • 特征：

      • 多路性

      • 独立性

      • 及时性

      • 交互性（通过终端与系统人机对话） 广泛性（多方面服务）

  • 实时系统

    • 实时系统的类型

      • 工业/武器控制

      • 信息查询

      • 多媒体

      • 嵌入式

    • 实时任务的类型

      • 周期性实时任务和非周期性实时任务

        • 周期性（周期的发送激励信号给计算机 要求按照制定周期循环执行）

        • 非周期性（联系截止时间 1⃣️开始截止时间2⃣️完成截止时间）

        • 硬实时任务和软实时任务

      • 实时系统和分时系统的特征比较

        • 多路性：实时系统的多路性指的是系统周期性的对多路现场信息采集

        • 独立性：互不干扰

        • 及时性：分时系统1-3s / 实时系统毫秒级（音乐/电视）

        • 交互性：分时系统可以多种服务 / 实时系统仅限于特定命令

        • 可靠性：可靠性

    • 单用户单任务操作系统

      • CP/M

      • windows

    • 多用户多任务操作系统

操作系统的基本特性 P13

  • 并发（Concurrence）

    • 并行：两个或者多个事件在同一时刻发生

    • 并发：两个或者多个事件在同一时间间隔发生（单处理机系统 每刻一道程序）

    • 进程：在系统中独立运行并作为资源分配的基本单位 （能独立运行的实体）

    • 由 一组堆栈 一组机器指令 数据 组成

  • 共享（资源复用）（在内存中）

    • 互斥共享方式 - 提出请求 若空闲分配 否则就等待 （临界资源，独占资源）

    • 同时访问方式

      • 允许多个进程访问 比如磁盘

  • 虚拟

    • 时分复用技术

      利用某设备为一用户服务的空闲时间为其他用户服务

      • 虚拟处理机技术 利用多道程序设计技术 每道程序至少一个进程 多道程序并发 虽然只有一台处理机 但可以实现为多个用户服务 一台物理上的处理机 虚拟为 多台逻辑上的处理机

      • 虚拟设备技术 一台物理设备 虚拟为 多台逻辑上的I/O设备

    • 空分复用技术 用存储器的空闲空间分区域存放和运行其他多道程序 提高内存利用率

  • 异步

    因为多个进程并发执行，当正在执行的进程提出对某种资源要求（临界资源） 而此时临界资源正在为其他进程服务 其必须等待 释放资源 直到资源空闲归还 所以因为资源等因素的限制，往往不能 “一气呵成” 必须 “走走停停” 进程以人们不可预知的速度向前推进，此即进程异步性

操作系统的主要功能 P16

操作系统为了多道程序的运行提供良好的运行环境

  • 处理机管理功能

    处理机的分配和运行以进程为基本单位

    • 进程控制 多程序环境下为使作业并发执行 必须为每道作业创建一个或几个进程 并分配资源 进程结束 撤销进程 回收资源 控制进程在运行过程中的状态转换

    • 进程同步

    • 进程互斥：对临界资源

    • 进程同步 当进程合作完成公共任务时，由同步机构对其次序加以协调 最简单的方法：对临界资源配置一把锁

    • 进程通信 合作完成共同任务需要交换信息 进程通信实现了相互合作的进程之间的信息交换 通过进程的消息队列

    • 进程调度

      • 作业调度 从后备队列选取若干作业，分配所需资源，分别建立进程，插入就绪队列

      • 进程调度 从进程就绪队列选出一个进程 分配处理机给他

  • 存储器管理功能

    • 内存分配

      主要任务：为程序分配内存空间；提高存储器利用率；
      允许正在运行的程序申请附加内存空间

      • 静态分配方式 作业装入时确定内存空间，不许作业在内存中移动或申请

      • 动态分配方式 允许继续申请新的空间，适应数据和程序的动态增长 允许移动

    • 内存保护 确保程序在自己的内存空间运行；不允许用户程序访问操作系统的程序和数据 不允许用户程序转移到非共享的其他程序中执行；（越界则中断请求，停止执行）

    • 地址映射 将地址空间中的逻辑地址转化为内存空间中对应的物理地址（硬件支持）

    • 内存扩充

      用虚拟存储技术，逻辑扩大

      • 请求调入功能 仅仅装入部分程序和数据就能启动，需要再从磁盘调入

      • 置换功能 如果没有调入空间，就把先不用的程序和数据调到硬盘，腾出内存

  • 设备管理功能

    完成用户进程提出的 I/O 请求，分配设备，完成操作 提高 CPU 和 I/O 设备的使用率 提高 I/O 速度 方便用户使用 I/O

    • 缓冲管理 缓和 CPU 和 I/O 设备速度不匹配的问题

    • 设备分配

    • 设备处理 又称 设备驱动程序：实现CPU和设备控制器之间的通信

  • 文件管理功能

    • 文件存储空间的管理 为每个文件分配必要的外存空间，提高外存利用率，进而提高文件系统的存取速度 需要相应的数据结构来记录文件存储空间的使用情况，以供分配存储空间时参考 有分配和回收功能

    • 目录管理 文件建立目录项 加以组织 只需要在外存保留一份该共享文件的副本

    • 文件的读/写管理和保护

      • 文件的读/写管理 读写不同时 可以和用指针；读写完成便修改指针

      • 文件保护 防止未经核准的用户存取文件；防止冒名顶替存取文件；防止以不正确的方式使用文件

  • 操作系统与用户之间的接口

    • 用户接口

      • 联机用户接口 终端命令

      • 脱机用户接口 用作业控制语言JCL把控制和命令写在作业说明书上，一起给系统

      • 图形用户接口

      • 程序接口 由一组系统调用组成，每个系统调用都是一个能完成特定功能的子程序

  • 现代操作系统的新功能

    • 系统安全

      • 认证技术

      • 密码技术

      • 控制访问技术

      • 反病毒技术

    • 网络的功能和服务

      • 网络通信

      • 资源管理

      • 应用互操作

    • 支持多媒体

      • 接纳控制功能

      • 多媒体文件的存储

OS结构设计 P22

  • 传统操作系统结构

    无结构操作系统 仅仅是一组过程的集合 每个过程可以任意相互调用其他 非常复杂混乱

    • 模块化操作系统

      • 基本概念

        • 模块独立性

          • 内聚性 模块内各部分间的紧密程度 内聚性越高 独立性越强

          • 耦合度 模块间相互联系相互影响的程度 耦合度越低 独立性越好

      • 模块接口法的优缺点

        • 优点 提高OS设计的正确性、可理解性、可维护性 增强OS可适应性 加速OS开发

        • 缺点 设计时的接口规定很难满足模块设计完成后对接口的实际需求 设计阶段需要作出很多决定，每个决定必须建立在上一个决定的基础上 模块化结构设计中，各模块齐头并进设计，很难每一步决定都建立在可靠基础上 模块-接口法 又被称为 无序模块法

      • 分层式操作系统

        • 基本概念 为了让模块接口法中的无序变得有序，引入有序分层法 在目标系统An和逻辑系统调用A0之间铺设若干层次软件 让目标系统通过中间若干软件层最终在A0上运行 常常采用自底向上法

        • 优缺点

          • 易保证系统正确性

          • 易扩充和易维护性

          • 系统效率低 单项依赖，必须每层之间都建立通信机制 一个功能穿越多个层次 通信开销大 导致效率降低

    • 客户/服务器模式

      • 客户/服务器模式的由来、组成和类型

        • 客户机 一个LAN网络上链接有多台客户机，客户进程在其上运行

        • 服务器 为所有网上的用户提供服务，被动等待客户机请求

        • 网络系统 链接所有客户机和服务器，实现通信和网络资源共享

      • 客户/服务器之间的交互

        利用消息机制通信，客户机和服务器共同完成对程序处理

        • 1.客户发送请求消息

        • 2.服务器接受消息

        • 3.服务器回送消息

        • 4.客户机接受消息

      • 客户/服务器模式的优点

        在分布式系统和网络环境下软件的主要工作模式之一

        • 1.数据的分布处理和存储

        • 2.便于集中管理

        • 3.灵活性和可扩展性 数量

        • 4.易于改编应用软件

    • 面向对象的程序设计技术简介

      • 面向对象技术的基本概念

        • 对象

        • 对象类

        • 继承

      • 面向对象技术的优点

    • 微内核OS结构

      有效支持多处理机运行，适用于分布式系统环境

      • 微内核操作系统的基本概念

        • 足够小的内核

          只将操作系统中最基本的部分放入微内核，如：

          • 与硬件处理紧密相关的部分

          • 一些较为基本的功能

          • 客户和服务器之间的通信

        • 基于客户/服务器模式 系统的绝大部分功能都放在微内核外的一组服务器中实现 线程运行在用户态，客户与服务器之间是利用微内核提供的消息传递机制实现交互

        • 应用“机制与策略分离”原理 机制：实现某一功能的具体执行机构（放在OS微内核） 策略：在机制的基础上借助某些参数和算法来执行功能的优化

        • 采用面向对象技术

      • 微内核的基本功能

        • 进程（线程）管理 进程调度 优先级队列 进程通信

        • 低级存储器管理

        • 中断和陷入处理

      • 微内核操作系统的优点

        • 提高系统的可扩展性

        • 增强了系统的可靠性

        • 可移植性强

        • 提供了对分布式系统的支持

        • 融入了面向对象技术

      • 微内核操作系统存在的问题

        • 性能很差

        • 文件系统放在核外
