图解密码技术笔记

C1 绪论

  • 部分用语

    • 明文（Plaintext）
    • 密文（Ciphertext）
    • 加密（Encrypt）
    • 解密（Decrypt）
    • 破译（Cryptanalysis）

  • 使用方法分类：确保机密性

    • 对称密码（Symmetric Cryptography）
      • 公共密钥密码（Common-Key Cryptography）
      • 私钥密码（Secret-Key Cryptography）
      • 传统密码（Conventional Cryptography）
      • 共享密钥密码（Shared-Key Cryptography）
    • 非对称密码（Asymmetric Cryptography）
      • 公钥密码（Public-Key Cryptography）
    • 混合密码技术（Hybrid Cryptosystem）
      • 结合了对称密码和非对称密码的优势

  • 其他密码技术

    • 单向散列函数（One-Way Hash Function）
      • 散列值 = 哈希值 = 密码校验和 = 指纹 = 消息摘要
      • 保证信息完整性
    • 消息认证码（Message Authentication Code）
      • 不仅确认篡改，还能确定是否来自期望的对象
      • 保证完整性+认证（Authentication）机制
    • 数字签名
      • 确保完整性+认证+防止否认（Repudiation）
    • 伪随机数生成器（Pseudo Random Number Generator，PRNG）
      • 如Web中进行SSL/TLS通信，生成仅用于当前通信的临时密钥

  • 隐写术与数字水印

    • 隐写术：消息嵌入，不能代替密码（加密+隐写）
      • 密码隐藏的是内容，隐写术隐藏的是消息本身
    • 数字水印：将著作权拥有者及信息隐写到文件的技术

  • 常识

    • 不用保密的密码算法
    • 低强度的密码比不加密更危险
    • 任何密码都会被破解
      • 严格说不会被破解的密码的存在性
        • 一次性密码本（One-Time Pad）
        • 量子密码
    • 密码只是信息安全的一部分

C2 历史上的密码

  • 凯撒密码（Caesar Cipherx）
    • 加密
      • 明文字母表平移，密钥为平移量
    • 解密
      • 暴力破解（Brute-Force Attack）
      • 穷举搜索（Exhaustive Search）
  • 简单替换密码（Simple Substitution Cipher）
    • 加密
      • 替换表作为密钥
    • 解密
      • 很难暴力破解，密钥（替换表的可能）太多（26!）
      • 频率分析，得出替换表
      • 高频低频字母/连续出现字母作为线索，越长越好破解
  • Enigma
    • 德语：谜
    • 加密
      • 转动的圆盘和电路实现的高强度密码，德国国防军采用
      • 国防军密码本作为密钥，用于设置Enigma
        • 1.用每日密码来设置Enigma
          • 2.加密发送者选择的通信密码，得到加密后的通信密码
        • 3.发送者选择的通信密码再设置Enigma（输入两次可以校验）
          • 4.加密消息
        • 5.拼接加密后的通信密码和消息
      • 每日密码作为加密密钥的密钥
      • 密钥加密密钥（Key Encrypting Key，KEK）
    • 解密
      • 1.分解密码
      • 2.利用每日密码设置Enigma
      • 3.解密通信密码
      • 4.利用解密的通信密码设置Enigma
      • 5.解密信息
  • 分开密码算法和密钥
    • 密钥为密码算法中可变的部分
    • 解决了希望重复使用，又不增加风险的问题

C3 对称密码

  • 编码：现实中的东西映射为比特序列的操作

  • XOR：异或，不同：1，同：0

    • A XOR B XOR B = A
      • B作为不可预测的比特序列作为随机数
    • 原始图像 + 蒙版 + 蒙版 = 原始图像

  • 一次性密码本（绝对不会被破译的运算）

    • 密钥配送问题，能配送为啥不送明文呢
    • 暴力也无法判断正确与否

  • DES（Data Encryption Standard）（Feistel网络）

    • 1977年美国联邦信息处理标准（FIPS）所采用的一种对称密码
    • 被美国以及其他国家的政府和银行广泛使用
    • 加密（一轮）
      • 1.左右等份
      • 2.右侧直接输入
      • 3.右侧+子密钥送入轮函数生成看似随机的比特序列
      • 4.比特序列与左侧XOR作为加密左侧
    • 解密
      • 相同，XOR的两次不变
    • 轮函数：根据右侧和子密钥进行加密生成比特序列

  • 差分分析与线性分析（Chosen Plaintext Attack，CPA 选择明文攻击）

    • 差分分析：改变部分明文观察密文如何改变
    • 线性分析：将明文和密文的对应比特进行XOR结果为0的概率是50%
    • 以AES为代表的现在分组密码算法已经考虑了针对这些分析的安全性

  • 三重DES（triple-DES，3DES）

    • 加密->解密->加密（为了可以兼容普通DES）向下兼容性
    • 如果所有密钥都相同，等价DES
    • 如果13同2不同，DES-EDE2
    • 如果123都不同，DES-EDE3
    • EDE（Encryption -> Decryption -> Encryption）
    • 解密：反过来走一遍
    • 现状：处理速度不高，除非需要特别向下

  • AES（Advanced Encryption Standard）

    • 取代DES成为新标准，有多个作为AES候选
    • 最终AES被确定为Rijndael
    • 竞选条件：无条件免费供全世界使用
    • 竞争实现标准化（standardization by competition）
    • 考虑弱点，算法速度，实现容易性
      • 不仅加密要快，密钥准备速度也要快

  • Rijndael

    • 分组密码算法
    • 分组长度以及密钥长度可以以32bit为单位在128b到256b范围选择
      • AES中只有128、192、256
    • 多个轮组成（没有用Feistal网络）
      • 每一轮为SubBytes、ShiftRows、MixColumns、AddRoundKey
    • SubBytes
      • 输入分组为128比特，16x8bit，16字节
      • 利用一张有256值的替换表（S-Box）进行替换
      • 以字节为单位进行处理？
    • ShiftRows
      • 按照一定规则进行行平移后输出
    • MixColumns
      • 对列进行矩阵运算后输出
    • AddRoundKey
      • 与轮密钥进行XOR
    • Feistal网络每一轮只加密一半输入
      • Rijndael可以在不同轮分别以不同的单位进行并行计算
    • 可能要重复10-14轮
    • 解密
      • 轮密钥XOR、逆矩阵运算、反向平移、逆替换
    • 破译
      • 计算过程可以全部用公式表达
      • 因此为新的攻击方法提供可能
      • 目前还没有有效攻击（2016）
    • 密码技术研究与评估委员会
      • CRYPTREC

C4 分组密码的模式——分组密码的迭代（伏笔）

  • DES和AES
    • 都属于分组密码，只能加密固定长度的明文
    • 要加密任意长度的明文就需要对分组密码进行迭代：分组密码的“模式”
  • 密码算法可分为分组密码和流密码
    • 分组密码
      • 每次只能处理特定长度数据的一类密码算法
      • 特定长度数据被称为分组
      • DES 64bit
      • AES 128bit
    • 流密码
      • 对数据流进行连续处理的一类密码算法
      • 1bit、8bit、32bit等单位进行加密和解密
    • 区别
      • 分组密码处理完一个分组结束，不需要记录加密进度
      • 流密码对一串数据进行处理，需要保持内部状态
  • 分组密码算法的迭代（分组密码的模式 mode）
    • ECB模式：Electronic CodeBook mode、电子密码本模式
    • CBC模式：Cipher Block Chaining mode、密码分组链接模式
    • CFB模式：Cipher FeedBack mode、密文反馈模式
    • OFB模式：Output FeedBack mode、输出反馈模式
    • CTR模式 ：CounTeR mode、计数器模式

  • 明文分组与密文分组
    • 明文分组
      • 分组密码中的加密对象，明文分组长度与分组密码算法的分组长度相等
    • 密文分组
      • 使用分组密码算法将明文分组加密之后所生成的密文

C5 公钥密码 ⭐️

  • 投币寄物柜

    • 有硬币，任何人都可以关闭寄物柜（硬币是关闭的密钥）
    • 有钥匙才能打开寄物柜 （钥匙是打开的密钥）

  • 密钥配送问题

    • 对称密码的密钥配送问题很难根本解决
    • 解决方案
      • 事先共享密钥
        • 局限于数量和便利程度
      • 密钥分配中心（KDC、Key Distribution Center）
        • 1.密钥分配中心通过伪随机数生成器生成一个会话密钥
          • 临时密钥
        • 2.KDC用A的密钥对会话密钥进行加密发给A
          • 同样操作给B
        • 3.A先用自己的密钥解密出会话密钥
        • 4.A再用会话密钥对邮件加密发送给B
        • 5.B再用自己的密钥解密出会话密钥
        • 6.B解密出邮件内容
      • Diffie-Hellman密钥交换
        • 加密通信的双方交换一定的信息，根据信息生成相同的密钥
        • 窃听者虽然能得到所交换的信息，但无法生成和双方相同密钥
      • 公钥密码
        • 发送者用加密密钥对通信内容进行加密并发送给接收者
        • 只有拥有解密密钥的人才能解密
        • 即对称密码的密钥配送问题可以用公钥密码来解决

  • 公钥密码 ⭐️⭐️⭐️⭐️⭐️

    • 前情提要
      • 发送者只需要加密
      • 接受者只需要解密
      • 解密密钥（私钥）不能被窃取
      • 加密密钥（公钥）可以被窃取
    • 公钥私钥一一对应，一堆公钥和私钥统称为密钥对（Key Pair）
      • 公钥进行加密的密文，必须用配对的私钥进行解密
      • 不能单独生成
    • 解决了密钥配送问题，但解决不了公钥认证问题
      • 判断所得到的公钥是否合法
      • 处理速度只有对称密码的鸡百分之一

  • 时钟运算

    • 加减乘、乘方：取mod
    • 除：如mod12 1/7=7
      • 7x7mod12=1
      • 在mod n的世界中，存在倒数的数字和n之间最大公约数是1
      • 即与n互质的数字
    • 离散对数
      • Diffie-Hellman密钥交换协议
      • EIGamal公钥算法就用了离散对数

  • RSA

    • 密文 = 明文^E mod N
    • E和N的组合就是公钥：公钥（E,N）or {E,N}
    • 明文 = 密文^D mod N
    • D和N的组合就是私钥：私钥（D,N）or {D,N}

  • 生成密钥对

  • 举例

    • 如
      • 1.p=17、q=19、N=17x19=323
      • 2.L = lcm(p-1,q-1) = lcm(16,18) = 144
      • 3.gcd(E,L)=1、E = 5，7，11，13，17，19，23，25…..
      • 4.E x D mod L = 1，D = 29（ 5 x 29 mod 144 = 1）
    • 此时
      • 公钥
        • E：5
        • N：323
      • 私钥
        • D：29
        • N：323
    • 明文
      • 比如小于N：323
      • 明文假设：123
    • 密文
      • 明文^E mod N
      • 123 ^5 mod 323 = 225
    • 明文
      • 密文^D mod N
      • 225 ^29 mod 323
      • = ( 225^10 mod 323 ) x ( 225^10 mod 323 ) x ( 225^9 mod 323 ) mod 323
      • 可用二进制提高效率

  • 对RSA的攻击

    • 离散对数问题没有高效算法
    • 对N进行质因数分解攻击
    • 推测p和q进行分析（如果伪随机数算法很差）
    • 求D与对N进行质因数分解在确定性多项式时间内等价
    • 中间人攻击
      • 认证：防御中间人攻击，确认公钥是否属于对方
        • 公钥的证书

  • 选择密文攻击

    • 假设知道密码算法但不知道密钥
    • 假设攻击者能使用服务：发送任意数据，服务器会当作密文解析并返回结果
      • 解密提示（Decryption Oracle）
      • 能抵挡选择密文攻击的算法强度很高
    • 对密文的“认证”
      • RSA-OAEP（Optimal Asymmetric Encryption Padding）
        • 最优非对称加密填充（明文前填充认证信息）
